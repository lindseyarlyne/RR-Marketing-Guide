Medium
Dashboard summary icons use accent gold + sage — inconsistent palette
AppPages.jsx → DashboardPage → SummaryCard icons
The three dashboard summary cards use #A8B5A0 (sage) for INCOME, var(--primary) for BILLS, and var(--accent) (warm gold) for AFTER BILLS. This mixes three hue families in one row. Simplify: use primary mauve for all three icons at different opacities, letting the label carry the semantic meaning.
Medium
Ritual page "daily runway" value uses var(--positive) sage green
AppPages.jsx → RitualPage
The "Your daily runway" value in the ritual summary card and the runway tile on the dashboard both use var(--positive) — which is sage green #7A9B78. This is a meaningful semantic color (money you have), but it conflicts with the "no green" palette direction. Suggest replacing with var(--primary) mauve for the runway figure, reserving green only if absolutely needed for a "safe" vs. "danger" comparison.

Critical

AppPages.jsx → BillsPage → paycheck card divs
The paycheck cards show only a mono label + serif italic amount + Jost sub-text. When no bills are assigned, the card shows "$0" with no context. 04
Copy & UX Language
4 issues
Critical
"Either" paycheck assignment label is ambiguous
AppPages.jsx → BillsPage (+ bill-redesign-rr.html already updated)
The paycheck assignment "Either" doesn't tell users whether it means "assigned to both," "will be split," or "not yet decided." Replace with "Flexible" — intentional, reads as a deliberate choice rather than an error state. Already applied in the bill redesign mockup; needs to flow into the actual app code.
Medium
"No paycheck assigned yet" is passive — should be actionable
AppPages.jsx → BillsPage → unassigned bill metadata
Bills with paycheck: null show "No paycheck assigned yet" in brick red — passive copy in an alarming color. Replace with: text "Not assigned" in muted-fg, no color alarm. If the user needs to act, a subtle nudge like a small indicator is better than red text on every unassigned row.
Medium
Sample data has duplicate phone bills — confusing onboarding
App.jsx → SAMPLE_DATA → bills array
The sample data contains both { name: 'Cell phone', amount: 70, paycheck: 2 } and { name: 'Phone bill', amount: 45, paycheck: null }. Two phone-related bills with different amounts is confusing for new users exploring the app. Remove one — keep "Cell phone" with an amount and keep the null-paycheck example as a different bill category (e.g., "Subscriptions").
Low
Bottom nav uses mono + letter-spacing at 10px — legibility issue on mobile
style.css → .bottom-nav button span
Mobile bottom nav labels use font-family: IBM Plex Mono, letter-spacing: 0.18em, font-size: 10px. At this size, the letter-spacing stretches "Settings" and "Ritual" uncomfortably. Remove the letter-spacing on the bottom nav specifically (keep it on the desktop tab strip where there's more space). The mono font alone at 10px is sufficient brand signal.

landing.css → .system-card
