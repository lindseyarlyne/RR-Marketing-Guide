What changed & why
01
Remove the dual indicator — keep only the check button
Each row had both an empty radio circle and a colored dot. The radio suggests "select one from a group" — but bills aren't mutually exclusive. The dot was meant to show paycheck assignment but had no legend. Fix: remove the bare radio, keep the check button for paid/unpaid toggling, and move paycheck identity into the metadata line as a small colored dot with text.
02
Demote "Shared" from inline spinner → read-only pill
An editable <select> inside a list row is a mis-placed control — ownership is a setting, not something you'd tap mid-scroll. Replace it with a small non-interactive pill tag. Editing ownership should happen inside a detail sheet opened by tapping the row.
03
Collapse 3 metadata lines → 1 dot-separated line
"Monthly", "Due on the 3rd", and "Assigned to Either paycheck" each took their own line, tripling row height. Combining them as "Monthly · Due 3rd · Either" in a single 10px IBM Plex Mono line reduces row height by ~55% while keeping all the same information visible.
04
Add utilization bar to paycheck cards
A "$0" paycheck card communicates nothing. Adding a 3pt progress bar (bills assigned / paycheck amount) with a "X% of $Y" label turns a passive label into an active budget signal. The Paycheck 2 bar uses the warning color to surface an imbalance worth addressing.
05
Accent "+ Add Bill" with primary color
The CTA was rendered in the same dim foreground color as disabled text, making it invisible against the dark card. Using var(--primary) — the same mauve already used on active tabs and links throughout the app — makes it clearly actionable with zero new vocabulary.
